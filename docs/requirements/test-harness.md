# Test Harness Baseline

### `TEST-UI-001`

macOS UI test target can launch the app successfully.

### `TEST-UI-002`

macOS UI test target captures launch performance metrics.

### `TEST-UI-003`

macOS launch test captures a launch screenshot artifact.

### `TEST-HARNESS-001`

Requirement-tag parsing accepts consecutive `// Requirement(s): ...` lines immediately adjacent to a test declaration (above or below) and merges all tagged IDs, allowing grouped lines (for example API IDs on one line and UX IDs on the next).


## Platform Applicability

- `TEST-UI-001`: macOS (required), iOS (planned), TUI (N/A).
- `TEST-UI-002`: macOS (required), iOS (planned), TUI (N/A).
- `TEST-UI-003`: macOS (required), iOS (planned), TUI (N/A).
